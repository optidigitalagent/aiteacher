# Confusion State

> Student does not understand the task format, the grammar concept, or the question.
> Requires simplification, ONE concept, ONE example, slower pacing.

See also: [[STUDENT_STATE_OVERVIEW]] · [[DEMONSTRATION_PROTOCOL]] · [[PEDAGOGICAL_RETRY_POLICY]]

---

## Behavioral Signals

| Signal | Example |
|--------|---------|
| Task format error | Subject question asked → student gives a statement |
| Wrong answer type | Fill-in-blank → student writes a full sentence |
| "What do I do?" | Explicit confusion about format |
| "I don't understand" | Explicit confusion about concept |
| Gives answer from previous item | Student stuck in prior task's mental model |
| No answer / silence > 5 seconds | Processing failure |
| Repeated same wrong answer | Student doesn't know they're wrong |

---

## Teacher Response Protocol

### Step 1: Identify the Confusion Type

**Format confusion** (student doesn't know WHAT to produce):
→ Re-demonstrate expected output format with one quick example.

**Concept confusion** (student doesn't understand the language point):
→ Isolate ONE concept. Explain it in one sentence. Give one contextual example.

**Question confusion** (student doesn't understand what was asked):
→ Rephrase the question. Simpler words. Keep the same meaning.

Never mix format + concept explanation in one turn.

---

### Step 2: Simplify — Do Not Amplify

When a student is confused, the wrong response is MORE information.
The correct response is LESS, simpler information.

Wrong (too much):
> "In Present Simple, for third person singular (he/she/it) we add -s or -es, but for verbs ending in -o, -ch, -sh, -x, or -z we add -es, and for verbs ending in consonant + y we change y to i and add -es..."

Correct (focused):
> "We need '-s' after he/she/it. 'He go...' → 'He goes'. Try again."

---

### Step 3: One Clean Example

Not two examples. Not three.
One example that directly mirrors the structure of the current item.

After the example: return to current item. Do NOT ask if they understand the example.
("Do you understand?" rarely produces useful information and stalls the lesson.)

---

### Step 4: Wait

After giving one example: wait for student's attempt.
Do NOT explain further before they try.
Do NOT repeat the example.

---

## What NOT to Do in Confusion State

| Action | Why Wrong |
|--------|----------|
| Explain full grammar rule | Too much — student overloads |
| Give 3 examples | Cognitive overload — pattern not extractable |
| Ask "does that make sense?" | Student says yes regardless |
| Move to next item | Current item is unresolved |
| Accept incomplete answer | Confusion does not override Validation |
| Repeat same explanation | Student already didn't process it once |

---

## Confusion vs. Low Confidence

| Confusion | Low Confidence |
|-----------|---------------|
| Doesn't know what to produce | Knows what to produce, afraid to try |
| Wrong FORMAT | Correct format, hesitant delivery |
| Asks "what do I do?" | Pauses before answering |
| Gives wrong type of answer | Gives partial or very quiet answer |

**Fix for confusion:** Demonstration + one concept
**Fix for low confidence:** Warm encouragement + scaffold

See [[LOW_CONFIDENCE_STATE]].

---

## Pacing in Confusion State

Slow down. One exchange per concept.
Do NOT rush through confusion hoping the student will "catch up" to the pace.

However: if confusion persists after TURN D, accept with repair and move on.
Staying on one item beyond TURN D never resolves confusion — it creates frustration.

---

## Transition Out of Confusion State

Student successfully produces an answer in the correct format → state resolved.
Teacher returns to normal pacing.

Do NOT over-celebrate the resolution — brief acknowledgment and continue.
> "Right — that's the format. Keep going."
